# MIGRATION.md — v3 Restructure (2026-05-18)

This document records every retirement from the v3 restructure. **Nothing has been physically moved or deleted yet** — this session ran without a bash sandbox. The cleanup script `scripts/cleanup-legacy.ps1` will perform the moves when Albert runs it on Windows. Review this file before running the script.

## How to execute

```powershell
# From the repository root:
cd "E:\Application downloads\Value"

# Dry run (default — prints what WOULD move):
powershell -ExecutionPolicy Bypass -File .\scripts\cleanup-legacy.ps1

# Actually move files:
powershell -ExecutionPolicy Bypass -File .\scripts\cleanup-legacy.ps1 -Apply
```

The script:
1. Creates `legacy/` subdirectories
2. Moves every retired path listed below to its archive destination
3. Deletes regeneratable junk (node_modules, __pycache__, .next, dist, .venv)
4. Logs every operation to `logs/cleanup-legacy-<timestamp>.log`
5. Never overwrites existing files in `legacy/` — fails loudly if a target already exists

## After running cleanup-legacy.ps1

Verify with:

```powershell
# Confirm legacy/ exists and is populated
Get-ChildItem .\legacy\ -Recurse -Depth 1 | Format-Table

# Confirm node_modules is gone
Test-Path .\apps\web\node_modules    # should be False (apps/web itself is in legacy/)

# Confirm canonical dashboard exists
Test-Path .\apps\dashboard\index.html  # should be True

# Confirm vault is untouched
(Get-ChildItem .\raw\ -Recurse -File).Count  # should match pre-restructure count
```

If anything is unexpected, the original folders are still in `legacy/` and can be restored with `Move-Item .\legacy\<thing>\ .\<thing>\`.
