# Formatting — VnEconomy-style intelligence letter

The editorial container for a daily intelligence report. Matches `news-dashboard-v2.html` styling (Roboto + Roboto Slab editorial, paper background, drop-cap on opener, condensed numerics).

## Template

```markdown
---
id: report-YYYY-MM-DD-NNN
type: report
kind: daily-brief
date: YYYY-MM-DD
title: <one-line dek that survives email subject lines>
subtitle: <optional second line>
sources: [...]
entities: [...]
themes: [...]
industry: ...
signal_score: ...
novelty_score: ...
prompt_version: ...
model: ...
template_version: intelligence-letter@v2
generated_at: <ISO timestamp>
mini_report:
  topic_sentence: ...
  bullets: [...]
---

# {title}

*{subtitle if present, in italic dek style}*

{topic_sentence as a normal paragraph — this is the lede}

{bullets, rendered as `<b>label:</b> body` list items}

---

**Source:** [{source_publication}]({url}) — {author}, {published_at}
**Related:** {related_reports as wikilinks}
**Signal:** {signal_score} · **Novelty:** {novelty_score} · **Confidence:** {confidence}

> *Generated by `{prompt_version}` on `{model}`, template `{template_version}`. Rate this report on the dashboard.*
```

## Rendering rules

- Title: short, declarative, ~60 chars max for email subject compatibility
- Subtitle: optional contextual qualifier
- Lede: the topic sentence becomes the first paragraph; no separate H1 above it
- Bullets: 4 ideal, render as a markdown list — the dashboard converts to `<ul><li><b>label</b> body</li></ul>`
- Footer: every report cites source + carries scores + carries prompt/model attribution

## Reader-side rendering

`apps/dashboard/index.html` reads `generated/index.json` and reconstructs the card UI: the mini_report becomes the card body, scores become indicator chips, the prompt_version becomes the "Rate" button's submission payload.
